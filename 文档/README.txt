一、如何运行
--------------------------------------------------------------------------------

本项目需要通过本地服务器运行，不能直接双击打开 HTML 文件。

方法一：使用 Python 服务器（推荐）
    1. 确保已安装 Python 3.x
    2. 双击运行项目根目录下的 server.py
    3. 打开浏览器访问 http://localhost:8080

方法二：如果提示未安装 Python
    1. 请先安装 Python 环境
    2. 安装地址：https://www.python.org/downloads/
    3. 安装时勾选 "Add Python to PATH"
    4. 安装完成后双击运行 server.py


二、作品概述
--------------------------------------------------------------------------------

NovaCampus Service 是一款专为校园生活打造的综合性服务平台。

它基于HarmonyOS风格设计，采用现代化的玻璃态（Glassmorphism）视觉效果，
为师生提供便捷的校园服务入口。

核心特点：
    • 简洁美观的 HarmonyOS 风格界面
    • 玻璃态视觉效果设计
    • 响应式布局，支持手机和桌面端
    • 无需安装，通过浏览器即可使用


三、能解决的问题
--------------------------------------------------------------------------------

1. 信息分散问题
   校园服务入口分散，学生需要记住多个网站或应用。
   本平台整合多种功能于一体，一站式访问。

2. 跨平台访问需求
   不受设备和系统限制，有浏览器就能使用。
   手机、平板、电脑均可正常访问。

3. 常用信息快速获取
   作息时间表、校园地图、失物招领等高频功能一键访问。

4. 校园社区需求
   提供学生交流论坛，方便讨论学习和校园生活话题。

5. 匿名反馈渠道
   通过匿名建议箱，学生可以放心提出意见和反馈。


四、功能模块
--------------------------------------------------------------------------------

1. 作息时间表 - 查看全校作息安排
2. 校园地图 - 快速定位校园设施位置
3. 失物招领 - 发布和查找遗失物品
4. 匿名建议箱 - 提出食堂、卫生、设施等方面的建议
5. 作业汇总 - 查看各年级各科作业
6. 班级相册 - 浏览班级活动照片
7. 校园电话 - 快速拨打校内常用电话
8. 活动日历 - 查看校园活动和大扫除安排
9. 社区论坛 - 学生交流讨论区
10. 关于我们 - 平台介绍和联系方式


五、技术架构
--------------------------------------------------------------------------------

前端技术：
    • HTML5 + CSS3 + JavaScript
    • 玻璃态（Glassmorphism）设计风格
    • HarmonyOS 图标库
    • 响应式布局设计

后端：
    • Python SimpleHTTPServer（开发阶段使用）
    • 可扩展为 Flask/Django/FastAPI 后端

数据存储：
    • JSON 文件存储（当前方案）
    • 可扩展为数据库存储


六、开发流程
--------------------------------------------------------------------------------

1. 需求分析
   确定目标用户群体和核心需求

2. 原型设计
   规划页面结构和功能模块

3. UI 设计
   采用 HarmonyOS 风格，制定配色和布局方案

4. 前端开发
   实现页面结构和样式，添加交互逻辑

5. 功能实现
   开发各个功能模块的详细逻辑

6. 测试优化
   在不同设备和浏览器上进行测试

7. 部署上线
   配置服务器并进行部署


七、应用反思
--------------------------------------------------------------------------------

优势：
    • 界面美观，符合现代审美
    • 多端适配，体验一致
    • 功能实用，直击痛点
    • 无需安装，使用便捷

不足：
    • 当前为前端静态版本，数据无法持久化
    • 缺乏用户认证系统
    • 社区功能较为基础
    • 缺少后台管理系统

可改进方向：
    • 添加后端支持，实现数据持久化
    • 接入真实天气 API
    • 增加用户登录和个人中心
    • 完善社区功能（发帖、评论、点赞）
    • 开发配套管理后台
    • 添加更多智能化功能
